**src/lib/parse-file.ts** -- Main-thread file handler (follows RESEARCH.md worker integration pattern):

1. Module-scoped `let worker: Worker | null = null` with lazy initialization via `getWorker()` that creates `new Worker('/worker.js')`.
2. `handleFileUpload(file: File): Promise<void>`:
   - Get store state via `useActivityStore.getState()`
   - Quick pre-check: if `!file.name.toLowerCase().endsWith('.fit')`, call `setError("That's not a FIT file, chief. We need the goods from your watch.", "Expected .fit file, got: {filename}")` and return
   - Call `store.startLoading()`
   - Read file as ArrayBuffer: `const buffer = await file.arrayBuffer()`
   - Set up `worker.onmessage` handler typed as `MessageEvent<WorkerResponse>`:
     - On `type: 'success'`: call `store.setActivity(response.activity)`
     - On `type: 'error'`: check if `response.details?.startsWith('SPORT_MISMATCH:')` -- if so, extract sport name and call `setError("Nice {sport}, but we're a running crew.", "FIT file contains {sport} activity. Only running activities are supported.")`; otherwise call `setError(response.message, response.details)`
   - Set up `worker.onerror` handler for unexpected worker crashes: `store.setError("Something went wrong during parsing.", error.message)`
   - Transfer the ArrayBuffer to worker: `worker.postMessage({ type: 'parse', buffer } satisfies WorkerRequest, [buffer])`
3. Export `handleFileUpload`.

**src/app/App.tsx** -- Root component with state-driven view switching:

1. Import all components: DropZone, BootSequence, DashboardSkeleton, HeaderBar
2. Import handleFileUpload from '../lib/parse-file'
3. Import useActivityStore
4. Read `status` from store
5. Use a local state `showBoot` (boolean) to manage the boot sequence timing:
   - When status transitions to 'loading', set showBoot = true
   - BootSequence's onComplete sets showBoot = false (the real parse may finish before the boot animation -- that's fine, the boot sequence is decorative)
6. View switching logic:
   - DashboardSkeleton renders ALWAYS as the background layer (it's the ghosted wireframe that's always visible behind everything)
   - When `status === 'empty'`: DropZone overlays on top (full-screen), passing handleFileUpload as onFile
   - When `status === 'loading' && showBoot`: BootSequence overlays on top (dissolve transition FROM drop zone)
   - When `status === 'loaded'`: HeaderBar appears at top, DashboardSkeleton fills in with real stats (drop zone and boot are gone)
   - When `status === 'error'`: DropZone overlays again (it renders ErrorDisplay internally based on store error state)
7. Use CSS transitions for the dissolve effect: drop zone fades out (opacity 0, pointer-events none), dashboard fades in. Use CSS classes driven by status.

**src/app/App.css:**
- Layer management: DashboardSkeleton at z-index 0, DropZone at z-index 50, BootSequence at z-index 60, HeaderBar at z-index 40
- Dissolve transitions: `.fade-enter`, `.fade-exit` with opacity transitions (~300ms)
- Ensure BootSequence covers the full screen during its animation

**src/app/index.tsx** -- Update to render App:
- Replace the placeholder div with `<App />`
- Import App from './App'
