<task type="auto">
  <name>Task 1: Install dependencies and create type contracts</name>
  <files>package.json, bun.lock, src/types/activity.ts, src/types/worker-messages.ts, src/lib/fit-types.d.ts</files>
  <action>
1. Install all Phase 1 dependencies:
   ```bash
   bun install @garmin/fitsdk react react-dom zustand downsample
   bun install -d @types/react @types/react-dom
   ```

2. Create `src/types/activity.ts` with these exact type definitions:
   - `ActivityRecord`: Individual data point with timestamp (Date), heartRate (number | null), cadence (number | null), speed (number | null), altitude (number | null), distance (number | null), positionLat (number | null), positionLong (number | null)
   - `SummaryStats`: totalDistance (number), totalDuration (number), totalTimerTime (number), avgSpeed (number | null), maxSpeed (number | null), avgHeartRate (number | null), maxHeartRate (number | null), avgCadence (number | null), totalAscent (number | null), totalDescent (number | null)
   - `ActivityMetadata`: device (string), date (Date), sport (string), duration (number)
   - `LapRecord`: startTime (Date), totalDistance (number), totalTimerTime (number), avgSpeed (number | null), avgHeartRate (number | null), avgCadence (number | null), maxHeartRate (number | null), maxSpeed (number | null)
   - `NormalizedActivity`: records (ActivityRecord[]), summary (SummaryStats), metadata (ActivityMetadata), laps (LapRecord[])

3. Create `src/types/worker-messages.ts` with the worker message protocol:
   - `WorkerRequest`: `{ type: 'parse'; buffer: ArrayBuffer }`
   - `WorkerResponse`: discriminated union of `{ type: 'success'; activity: NormalizedActivity }` and `{ type: 'error'; message: string; details?: string }`

4. Create `src/lib/fit-types.d.ts` -- TypeScript module declaration for `@garmin/fitsdk`:
   - Declare module '@garmin/fitsdk'
   - Export class `Stream` with static method `fromArrayBuffer(buffer: ArrayBuffer): Stream`
   - Export class `Decoder` with: constructor(stream: Stream), static `isFIT(stream: Stream): boolean`, static `checkIntegrity(stream: Stream): boolean`, method `read(options?: DecoderOptions): DecoderResult`
   - `DecoderOptions` type with boolean fields: applyScaleAndOffset, expandSubFields, expandComponents, convertTypesToStrings, convertDateTimesToDates, mergeHeartRates
   - `DecoderResult` type: `{ messages: FitMessages; errors: Error[] }`
   - `FitMessages` type with: fileIdMesgs (any[]), sessionMesgs (any[]), recordMesgs (any[]), lapMesgs (any[]), deviceInfoMesgs (any[]), and index signature `[key: string]: any[]`
   - Keep it focused -- only the exports we actually use. The message arrays use `any` because Garmin provides no types for the decoded field shapes.

All types use `export type` (not `export interface`) to satisfy verbatimModuleSyntax in tsconfig.
  </action>
