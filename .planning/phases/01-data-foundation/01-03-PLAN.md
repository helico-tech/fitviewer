**DropZone.tsx** -- Full-screen takeover drop zone per CONTEXT.md locked decisions:

1. Component accepts `onFile: (file: File) => void` prop.
2. Renders as a full-screen fixed overlay (`position: fixed; inset: 0; z-index: 50`).
3. Contains:
   - Inviting headline: "Drop your FIT file to launch telemetry" (or similar)
   - Brief tagline below it
   - File picker button as fallback -- creates a hidden `<input type="file" accept=".fit">`, clicks it programmatically on button press
   - When store status is 'error', renders `<ErrorDisplay />` component inline (not a modal -- errors shown in-place on the drop zone itself)
4. Drag-over behavior:
   - Uses native HTML5 DnD: onDragOver, onDragLeave, onDrop handlers on the container div
   - On drag enter/over: adds a CSS class `drop-zone--active` that triggers the "power on" animation
   - On drag leave/drop: removes the class
   - The "power on" effect: the ghosted dashboard wireframe behind the overlay starts lighting up (use CSS transitions on opacity/filter/box-shadow). The overlay itself may brighten or shift. This should feel like a system activating, not just a border color change.
5. Drop handler: prevents default, extracts the first file from dataTransfer.files, calls onFile(file).
6. Use useCallback for all handlers to prevent re-renders.

**DropZone.css:**
- `.drop-zone`: Fixed full-screen, dark semi-transparent backdrop with subtle gradient, flexbox center content, transition on all visual properties
- `.drop-zone--active`: Backdrop lightens, glow effects appear (box-shadow with neon color, e.g., `0 0 60px rgba(0, 255, 170, 0.3)`), text brightens
- Use CSS custom properties for neon accent colors: `--neon-primary: #00ffaa;` `--neon-secondary: #00ccff;`
- Smooth transitions: `transition: all 0.3s ease`
- The file picker button: subtle outline style, neon accent on hover

**ErrorDisplay.tsx:**
1. Receives props: `message: string`, `details?: string`
2. Shows the playful error message prominently
3. Has an expandable "Show details" toggle (useState for open/close) that reveals technical details when present
4. No modal -- this renders inline within the drop zone
5. User can drop another file immediately without dismissing anything

**ErrorDisplay.css:**
- Error message in a warm/amber accent color (not harsh red -- this is playful, not alarming)
- Expandable details in a monospace font, dimmer color
- Smooth expand/collapse transition (max-height transition or similar)

**BootSequence.tsx** -- Sims-style fake loading per CONTEXT.md:

1. Component mounts and cycles through an array of fun/ridiculous status messages. Examples (Claude's discretion on exact content):
   - "Initializing telemetry systems..."
   - "Calibrating heart rate sensors..."
   - "Analyzing stride patterns..."
   - "Reticulating splines..."
   - "Decoding satellite uplink..."
   - "Charging flux capacitors..."
   Keep it to 6-8 messages.
2. Messages cycle rapidly (every 200-300ms) using useEffect with setInterval.
3. After a fixed total duration of ~1.5 seconds, calls an `onComplete` callback prop.
4. Visual: dark background, neon-colored terminal-style text (monospace), maybe a blinking cursor effect. Each message appears one at a time (not accumulated). Optional subtle fade-in per message.
5. Should feel like a sci-fi boot sequence -- quick and fun, not a real progress indicator.

**BootSequence.css:**
- Full-screen centered, dark background matching the app
- Monospace font for messages, neon green or cyan color
- Optional: scanline overlay effect or subtle text-shadow glow
- Fade-in animation per message

**DashboardSkeleton.tsx:**

1. Reads from the activity store: `status` and `summary`.
2. When `status` is 'empty' or 'loading': renders a ghosted wireframe layout. This is a visual tease of what the Phase 2 dashboard will look like:
   - Summary stats row at top (placeholder boxes for distance, duration, pace, HR, cadence, elevation)
   - Placeholder chart areas (3-4 rectangular blocks representing future pace/HR/elevation/cadence charts)
   - Placeholder map area (rectangular block)
   - Placeholder lap table area
   All rendered as dim, ghosted rectangles/outlines with very low opacity (`opacity: 0.1-0.15`).
3. When `status` is 'loaded': the summary stats row fills in with real data from `summary`:
   - Distance: format as km with 2 decimal places (totalDistance / 1000)
   - Duration: format as H:MM:SS or M:SS from totalTimerTime
   - Avg Pace: derive from avgSpeed as min/km (1000 / avgSpeed / 60 for minutes, then format MM:SS)
   - Avg HR / Max HR: display as bpm, show "--" if null
   - Cadence: display as spm, show "--" if null
   - Elevation Gain: format as meters with 0 decimals, show "--" if null
   The chart/map/lap placeholders remain ghosted (those fill in during Phase 2).
4. Each stat card: label on top (small, dimmed), value below (large, bright neon).
5. Use a helper function for formatting (keep it in the component file or a small utils -- Claude's discretion).

**DashboardSkeleton.css:**
- CSS Grid layout for the dashboard wireframe
- Stats row: flex or grid, evenly spaced stat cards
- Ghosted placeholders: `background: rgba(255,255,255,0.03); border: 1px solid rgba(255,255,255,0.06);` with rounded corners
- Filled stats: neon-colored values, subtle glow text-shadow
- Transition from ghosted to filled: opacity/color transition

**HeaderBar.tsx:**

1. Reads `metadata` and `status` from the activity store.
2. Only renders when `status === 'loaded'`.
3. Displays in a single row: "Garmin FR965 . Mar 10, 2026 . Running . 42:15" format (dot-separated metadata).
   - Device: metadata.device
   - Date: format metadata.date as "MMM DD, YYYY"
   - Sport: capitalize metadata.sport
   - Duration: format metadata.duration as MM:SS or H:MM:SS
4. Has a "Replace" button on the right side that calls store.reset() to go back to the drop zone.
5. Fixed to top of viewport, slim height (~48px), dark background with subtle bottom border.

**HeaderBar.css:**
- Fixed top, full width, flexbox row with space-between
- Metadata text: small, dim white, dot separators
- Replace button: small, outline style, neon accent on hover
- Subtle bottom border or box-shadow separator
